Operations
    Operations may or may not take in up to one immediate value, which is either 8 or 16 bits depending on the instruction.
    If the virtual immediate flag is set, the immediate value is sourced from the immediate value register (IM) instead of being read from the instruction stream.
    An instruction can be one to three bytes long. The same operation does not always take the same number of bytes, as when the virtual immediate flag is set,
    the immediate value is sourced from the IM register and not read from the instruction stream. This can be strange
    for a beginner, but allows very compact code.
    When VF is set, the next instruction that takes an immediate value (n.b. nullary instructions excluded) will use the value in the IM register and clear the VF flag. 

Display format:
    128x128 pixels, 255 colors (8-bit color, RRRGGGBB)
    Display buffer is 128 * 128 = 16384 bytes (16k) at any memory location
    Frame rate is on a static 60fps timer, but the display is refreshed manually
    a frame can be waited for with WAIT

Types:
    8-bit unsigned integers (0-255)
    16-bit unsigned integers (0-65535)
    8-bit signed integers (-128 to 127)
    16-bit signed integers (-32768 to 32767)

    64-bit audio voice
        bit 0-15 = wave ptr
        bit 16-31 = frequency
        bit 32-47 = phase
        bit 48-51 = volume (0-15)
        bit 52-55 = pan (0-15, where 0 is left and 15 is right)
        bit 56 = enabled flag, if this bit is not set the voice is not played
        bit 57 = loop flag, if this bit is set the voice will loop back to the start of the wave when it reaches the end
        bit 58 = pingpong flag, loop must be enabled for this to work, and if this is set the voice will play forwards then backwards when it loops, instead of just jumping back to the start of the wave
        bit 59 = noise flag, will play determinstic noise using the wave buffer as the source of the noise pattern
        bit 60 = reset flag, if this bit is set the voice will reset its phase to 0 when it is triggered to play
        bit 61-63 = reserved for future use, should be set to 0

Waveform
    64 samples of 8-bit signed audio data, which can be used as a waveform for the audio voices. The waveforms are stored arbitrarily in memory, and the location of the waveforms is specified by the wave ptr.
    Waveform buffer is 64 bytes per waveform, and there can be any number of waveforms in memory.
Voice
    There are 8 audio voices after the audio pointer (AP) in memory. It is safe to leave unused voices as all zeroes, as this will leave the "enabled" bit unset.
    The voice buffer is 8 voices * 8 bytes per voice = 64 bytes total for all voices.
Sample rate
    22050 Hz is plenty
Memory Map (required):
    "Registers" (which are reserved memory locations):
    PC = [0x00 - 0x01] = program counter, points to the next instruction to execute, initialized to wherever the code starts. a sane default would be 0x10
    SP = [0x02 - 0x03] = start of stack, its freeform like brainfuck.
    DP = [0x04 - 0x05] = start of display data, any data written in the section of memory after (up until the offset of the size of the pixel buffer) will be displayed on the screen when "rfd" (refresh display) is called
    AP = [0x06 - 0x07] = start of audio data, any data written in the section of memory after (up until the offset of the size of the audio buffer) will be played as audio when "rfa" (refresh audio) is called.
    IN = [0x08] = input register, each bit represents a different button
        bit 0 = up
        bit 1 = down
        bit 2 = left
        bit 3 = right
        bit 4 = A
        bit 5 = B
        bit 6 = start
        bit 7 = select
    FL = [0x09] = flags register, used for conditional jumps and other operations, bits can be set or cleared by instructions
        bit 0 = reserved
        bit 1 = carry flag (CF), set if an operation results in a carry out of the most significant bit
        bit 2 = reserved
        bit 3 = overflow flag (OF), set if an operation results in an overflow
        bit 4 = grow flag (GF), controls the growth direction of the stack, if this bit is set the stack grows upward, if it is clear the stack grows downward. This can technically be changed mid-execution, as we don't maintain internal structure for the stack.
        bit 5 = halt flag (HF), if this bit is set the VM will halt execution. This cannot be cleared by any instruction, it can only be set, because no instruction will be executed after it is set. In debugging, it may be useful to halt via an instruction (breakpoint) and resume via a special VM function
        bit 6 = virtual immediate flag (VF), if this bit is set, the immediate value register (IM) will be used as the source for the immediate value in the next instruction call.
    IM = [0x0A - 0x0B] = Immediate value register, 
Stack
    The stack grows with the direction determined by the grow flag (GF) in the flags register. If GF is set, the stack grows upward (towards higher memory addresses), and if GF is clear, the stack grows downward (towards lower memory addresses). 
    The stack pointer (SP)'s initial value is determined by the cartridge, and it is up to the program code to not over/underflow the stack. Wrapping arithmetic is used for the stack pointer, so if stack pointer manages to grow beyond the bounds of memory strange things will happen
    The stack is an untyped tape with no bounds or enforced structure. It is up to the program code to use it in a sensible way. 
    Element sizes are not fixed on the stack.
Operations
    at each step of execution, the VM reads the opcode at the memory location pointed to by the program counter (PC). The PC is byte-addressed, and the opcode is always 8 bits. Depending on the operation, a variable number of bytes are read as operands following the opcode. After the instruction is executed, the PC is incremented by the total number of bytes read (1 byte for the opcode plus any additional bytes for operands) to point to the next instruction.
    Alignment of stack and memory accesses is not enforced, so it is possible to read/write 16-bit values from/to odd memory addresses.

    All operations that are unassigned are to be treated as NOP. 0xA0 (control flow) is the canonical NOP, and is the only guaranteed NOP for future compatibility aside from special 0xA0.
    0x00 is reserved as a fail-safe NOP in production, but in debugging it can be used for debugging.

Cartridge
    the cartridge is a full ram snapshot (64kb). vital registers like the program counter and stack pointer are sourced from the cartridge, meaning the cartridge fully defines the initial state of the console.
    When the VM starts, the first instruction will be read from the memory location pointed to by the program counter (PC). if the PC is invalid in any way then undefined behavior will occur
    everything is little endian
